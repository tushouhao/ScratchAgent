## 指令

对指定代码做审查，检查正确性、安全性、可靠性、风格，给出分级建议。

## 步骤

1. 获取待审查代码：
   - 若用户指定了文件：用 read_file 读取
   - 若未指定：用 run_command 执行 `git diff HEAD` 获取未提交改动
2. 逐项检查（优先级：正确性 > 安全性 > 可靠性 > 风格）：
   - **正确性**：逻辑错误、边界遗漏、类型不匹配
   - **安全性**：注入风险、硬编码密钥、不安全 API
   - **可靠性**：未处理异常、资源泄漏、竞态条件
   - **风格**：命名、注释、代码组织
3. 分级列出发现：🔴 严重 / 🟡 建议 / 🟢 优良
4. 给出可操作的修改建议（若用户要求则直接改）。

## 验证

- 审查报告完整，覆盖所有改动文件。

## 示例

- `use_skill("review")` — 审查当前未提交改动
- `use_skill("review", "src/auth.js")` — 审查指定文件
